2 ## SnorkTracker
3  **GPS IoT tracker board for scanning gps and environment information and sending them to a MQTT server via GPRS.**
4

9 ### Description
10  The goal of this board is to have an easy-to-use, configurable GPS tracker that passes its data to an MQTT server.
11  It should also be able to scan other environmental data and operate on a 12V power supply with low power consumption
12  over a long period of time.
13 
14  The main components of the systems are:
15  * A 'Wemos D1 mini lite' to run the software.
16  * A BK-808 v2 (SIM808) GPS/GPRS modul to communicate with a MQTT server and to send/receive SMS.
17  * An BME280 modul to scan environmental data.
18  * An LM2596 DC-DC converter to supply the GPS/GPRS board.
19  * An MCP1703 for the 3.3V Wemos power supply.
20 
21  To interact with the board it has an easy-to-use browser interface to show the
22  scanned values, configure the system or to debug at runtime via console.
23  It can connect to an existing wlan or can work as an access point.
24 
25  The board has an resistor divider to scan the 'power supply voltage' and
26  a BME280 sensor to read the temperature, humidity and pressure.
27 
28  The module operates with a supply voltage of approx. 6V to 16V. Perfect for a car battery.
29 
30  For saving power the 'supply voltage' can be used to decide if the battery is on loading or not
31  and so the system can configured to go into deep sleep mode which uses only ~0.3mA.
32 
33  With the GSM chip, the system can interact with SMS commands to set default values or send current
34  information such as GPS or temperature.
35 
36  You can also use the software on a esp8266 chip like the wemos d1 for having a serial console over HTTP if you connect
37  to the D5 and D6 pin.

54 ### Setup Arduino Environment
55  * Download Arduino IDE from
56  https://www.arduino.cc/en/Main/Software
57  and follow the installation instructions.
58  * Install ESP8266 board software
59  * In Arduino IDE set 'File | Preferences' to Boards Manager URL
60  http://arduino.esp8266.com/stable/package_esp8266com_index.json
61  * Open 'Tools | Board... | Boards Manager...'
62  and search for 'esp8266' and install the software package
63  * Install the ESP8266 file-system up-loader from
64  https://github.com/esp8266/arduino-esp8266fs-plugin
65  and follow the installation instructions.
66  * Download **SnorkTracker** software by clone or download.
67  * Copy the content of the snorktracker/lib directory to the Arduino installation lib directory.
68  For example c:/Program Files (x86)/Arduino/lib/
69  * Configure Arduino IDE
70  * From 'Tools | Boards' choose 'LOLIN(WEMOS) D1 mini Lite'
71  * Change 'Tools | Flash Size' to '1M (64k SPIFFS)'
72  * Change 'Tools | IwIP variant' to 'v14 Higher Bandwidth'

77 ### Assemble the Components
78  * Take the Breadboard and the other items from the part-list.
79  ![Figure 3](images/Breadboard.jpg "Figure 3")
80  * First of all unsolder the (ON/OFF) pin 5 from the DC-DC Modul, bend it up and
81  solder a line to it. After that put 4 spacers under the DC-DC Modul and solder it
82  to the breadboard. Check that the bottom of the LM2596 module is not in contact with the board.
83  See
84  ![Figure 4](images/DC-DC-Modul.jpg "Figure 4")
85  * Place the other parts from the part-list on the breadboard and solder it as shown.
86  Pay attention to the polarity of the capacitors!
87  Split the header connector to the length it is needed and solder it.
88  A header connector is needed because flashing of the Wemos modul often does not work
89  if it is still connected to the board.
90  Check if the Wemos and SIM808 modul has enough space and fits into the pins.
91  ![Figure 5](images/Unplugged.jpg "Figure 5")
92  * Connect all the pins of the parts as seen on the circuit image with wires.
93  ![Figure 4](images/Back.jpg "Figure 4")
94  * The final board could look like this
95  ![Figure 6](images/SnorkTracker.jpg "Figure 6")
96  * Check all the wiring with a multimeter. Also against short-circuit.

98 ### Setup the LM2596 voltage and install the software
99  * The first step we have to do is to set the right 5V for the SIM808 modul.
100  * For that unplug the Wemos and the SIM808 modul.
101  * Connect a 12 power supply to the screw connector. Be sure + and - are correct!
102  * Switch on the LM2596 modul by connection the unsoldered pin 5 to ground.
103  * Now, connect the multimeter to the output pins of the DC-DC modul and screw on
104  the modul resistor until the multimeter shows exact 5.0V on the two out-pins.
105  * Check the pins on the female header connector for the right voltage.
106  * Check the 3.3V on the corresponding WEMOS pin.
107  * Check the 5.0V pin of the SIM808 modul connector pin.
108  * Check the 3.3V on the BME280 modul. Be aware that the ground pin is connected to
109  the Wemos modul and still open.
110  * Now its time to flash the software to the Wemos modul.
111  * Unplug the Wemos modul (if plugged) and connect it with the USB cable to your computer.
112  After connect via usb you should can the select the arduino port under 'Tools | Port | ...'.
113  * Open the tracker.ino file in the tracker folder and check the Arduino environment settings.
114  * Upload the SPIFFS files with the 'Tools | ESP8266 Sketch data upload' menu.
115  The SPIFFS files (html's, styles and javascripts) are taken from the data subdirectory.
116  * After that go to the *Config.h* file of the project and enter the right configure values.
117  * Now we can flash the program to the Wemos chip by clicking 'Sketch | upload'
118  * Unplug and plug again the Wemos modul to the usb cable and start immediately the serial monitor
119  of the Arduino IDE 'Tools | Serial monitor'.
120  * If everything is correct you can see an access-point in the WLAN of your computer and you can connect
121  to the browser interface by calling the ip in your browser.
122  The software works alos in a captive way, so you can entering a dummy (not https) url and the system should you
123  redirect to the web interface. For example http://dummy.com
124  If this does not work find out the new ip via your WLAN router interface.
125  * If the software works fine on the usb cable then you can plug the wemos into the board and connect a power supply.
126  Check again the connectivity.

128 ## Function
129  * The software in the Wemos modul creates a web-server in station and/or in access-point mode.
130  * The analog input of the wemos is connected to the middle of two voltage divider resistors.
131  So it can read the power supply voltage of the system (for example the car battery voltage).
132  * The software can be configured to recognize a charging or discharging car battery to switch on or off the
133  submodules. With my car this was:
134  * 11.8 Volt if the car was switched off for more than a day.
135  * 14.2 Volt if the engine is running.
136  * 13.3 Volt after switch off the engine.
137  * A BME280 sensor is connected to the 3.3V power and can be switch on via setting the D4 pin of the wemos chip
138  to ground. This is done only from time to time to save energy.
139  * There is a LM2596 DC-DC modul on the board which can be switched on by a '-' signal to the pin 5 of the LM2596.
140  Behind the DC-DC modul there is a SIM808 modul with GPS/GPRS/GSM functionality. So the Wemos can switch on/off
141  the SIM808 chip to save energy.
142  * The Wemos chip can communicate with the SIM808 modul via RX and TX signal and AT commands.
143  * Via the GPRS modul it can send the scanned data to a MQTT server and can communicate via SMS to a phone.
